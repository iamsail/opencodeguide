# Product Context

## Problem Statement
OpenCode is a powerful tool but lacks approachable, scenario-based documentation. Official docs (hypothetically) might be dry or incomplete. New users, especially those sensitive to privacy or cost, struggle to set up local models (Ollama), understand how it differs from easy-to-use SaaS tools like Claude Code, or choose the best high-performance models for coding agents without complex configuration.

## Target Audience
1.  **English Users**: Developers, AI Engineers, CLI power users, privacy-conscious enterprise devs.
2.  **Chinese Users**: Developers facing network constraints, wanting low-cost local solutions, or needing中文 entry-level guides, and those looking for a curated model gateway (Zen).

## User Experience Goals
*   **Trustworthy**: The site should look professional (like official docs), not like a random blog.
*   **Fast**: Instant page loads.
*   **Clear**: Code blocks must be legible, copyable, and syntax-highlighted.
*   **Distinction**: Clear separation between English and Chinese content to avoid SEO confusion.

## Content Strategy
*   **Not Translation**: Chinese content is localized, not just translated (e.g., mentioning domestic mirrors).
*   **Hub Strategy**: Use authoritative hubs (e.g., `/en/oh-my-opencode`, `/en/mcp`) to capture high-level intents and distribute authority to deep-dive technical pages.
*   **Feature-Level Comparison**: Compare OpenCode vs Claude Code objectively (Privacy, Cost, Offline).
*   **Actionable**: Every guide ends with valid CLI commands or config snippets.
