# 专注模式 (Zen)

## 这是什么？
专注模式 (Zen Mode) 是 OpenCode 提供的一种极简交互体验，旨在最大程度减少干扰。在 Zen 模式下，不必要的 UI 元素会被隐藏，系统通知会被抑制，AI 仅在关键时刻请求用户确认，自动处理大部分常规任务。

## 什么时候你会用到它？
- 当你正在攻克一个复杂的算法问题，需要全神贯注思考，不希望被频繁的 UI 弹窗打断时。
- 当你给予 AI 一个明确且长期的任务（如“重构整个模块”），希望它自动执行直到完成或遇到阻碍时。
- 当你在小屏幕设备上工作，需要最大化代码可视区域时。

## 核心概念
- **自动确认**：对于低风险的文件读取或非破坏性命令，AI 将自动执行而无需用户按 `y` 确认。
- **极简 UI**：隐藏侧边栏、状态栏等非核心信息，只保留对话流和代码区。
- **错误自动修复**：遇到执行错误时，AI 会尝试自动分析并修复，减少人工干预。

## 常见误解 / 注意事项
- Zen 模式并不意味着“完全无人值守”，对于删除文件或高风险系统命令，OpenCode 依然会强制要求确认。
- 初学者建议先在普通模式下熟悉 AI 的行为模式后，再尝试 Zen 模式。
