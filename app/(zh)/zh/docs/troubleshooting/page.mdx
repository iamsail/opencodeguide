# 故障排除 (Troubleshooting)

**常见问题及解决方法。**

要调试 OpenCode 的任何问题，您可以查看其存储在本地的日志或会话数据。

## 日志 (Logs)

日志文件写入路径如下：

* **macOS/Linux:** `~/.local/share/opencode/log/`
* **Windows:** `%USERPROFILE%\.local\share\opencode\log\`

日志文件以时间戳命名（例如 `2025-01-09T123456.log`），系统会保留最近的 10 个日志文件。

您可以通过命令行选项 `--log-level` 设置日志级别以获取更详细的调试信息。例如：`opencode --log-level DEBUG`。

## 存储 (Storage)

OpenCode 将会话数据和其他应用程序数据存储在磁盘上的以下路径：

* **macOS/Linux:** `~/.local/share/opencode/`
* **Windows:** `%USERPROFILE%\.local\share\opencode`

该目录包含：

* `auth.json` - 身份验证数据（如 API 密钥、OAuth 令牌）。
* `log/` - 应用程序日志。
* `project/` - 特定项目的数据（如会话和消息数据）。
* 如果项目位于 Git 仓库内，数据存储在 `./<project-slug>/storage/`。
* 如果不是 Git 仓库，则存储在 `./global/storage/`。

## 常见问题 (Common issues)

以下是一些常见问题及其解决方法：

### OpenCode 无法启动

* 检查日志中的错误信息。
* 尝试运行 `opencode --print-logs` 在终端查看输出。
* 确保您使用的是最新版本，运行 `opencode upgrade` 进行升级。

### 身份验证问题

* 尝试在 TUI（终端界面）中使用 `/connect` 命令重新进行身份验证。
* 检查您的 API 密钥是否有效。
* 确保您的网络允许连接到服务商的 API。

### 模型不可用

* 检查您是否已完成该服务商的身份验证。
* 验证配置文件中的模型名称是否正确。
* 某些模型可能需要特定的访问权限或订阅。

如果您遇到 `ProviderModelNotFoundError`，通常是因为某处引用的模型格式不正确。模型的引用格式应为：`<providerId>/<modelId>`。
**示例：**

* `openai/gpt-4o`
* `openrouter/google/gemini-2.0-flash`
* `opencode/kimi-k1`

运行 `opencode models` 可以查看您有权访问的所有模型。

### ProviderInitError

如果您遇到 `ProviderInitError`，通常意味着您的配置无效或已损坏。
**解决方法：**

1. 首先，参考[服务商指南](/zh/docs/providers)确认服务商设置是否正确。
2. 如果问题依旧存在，尝试清除存储的配置：
```bash
rm -rf ~/.local/share/opencode
```


3. 在 TUI 中使用 `/connect` 命令重新连接服务商。

### AI_APICallError 和服务商包问题

如果您遇到 API 调用错误，可能是由于服务商软件包过时。OpenCode 会根据需要动态安装服务商包（OpenAI, Anthropic, Google 等）并进行本地缓存。
**解决方法：**

1. 清除服务商包缓存：
```bash
rm -rf ~/.cache/opencode
```


2. 重启 OpenCode 以重新安装最新的服务商包。
这会强制 OpenCode 下载最新版本的服务商包，通常可以解决与模型参数或 API 更改相关的兼容性问题。

### Linux 上无法复制/粘贴

Linux 用户需要安装以下剪贴板实用程序之一才能正常使用复制/粘贴功能：

* **对于 X11 系统：**
```bash
apt install -y xclip
# 或
apt install -y xsel
```


* **对于 Wayland 系统：**
```bash
apt install -y wl-clipboard
```


* **对于无头（Headless）环境：**
```bash
apt install -y xvfb
# 并运行：
Xvfb :99 -screen 0 1024x768x24 > /dev/null 2>&1 &
export DISPLAY=:99.0
```

OpenCode 会检测您是否在使用 Wayland 并优先选择 `wl-clipboard`，否则会按顺序查找 `xclip` 和 `xsel`。
