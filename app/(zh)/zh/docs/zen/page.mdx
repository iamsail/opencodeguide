# Zen

**OpenCode 提供的精选模型列表**

OpenCode Zen 是由 OpenCode 团队 provide 并经过测试与验证的模型列表。

> **注意：** OpenCode Zen 目前处于测试阶段。

Zen 的使用方式与 OpenCode 中的任何其他服务商（Provider）相同。您只需登录 OpenCode Zen 并获取 API 密钥即可。这是完全可选的，您无需使用它也能使用 OpenCode。

## 背景 (Background)

市面上模型众多，但只有少数能胜任“编程代理（Coding Agents）”的工作。此外，不同服务商的配置差异巨大，导致性能和质量参差不齐。

> **提示：** 我们测试并精选了一组能与 OpenCode 良好协作的模型和服务商。

如果您通过 OpenRouter 之类的工具使用模型，往往无法确定是否获得了该模型的最佳版本。为了解决这个问题，我们做了以下工作：

1. 测试了精选模型，并与模型团队沟通了最佳运行方案。
2. 与部分服务商合作，确保这些模型被正确部署。
3. 对模型和服务商的组合进行了基准测试（Benchmarked），整理出一份我们放心推荐的列表。

OpenCode Zen 就是一个让您能够访问这些精选模型的 AI 网关。

## 运作方式 (How it works)

OpenCode Zen 的使用非常简单：

1. 登录 OpenCode Zen，添加账单信息并复制您的 API 密钥。
2. 在 TUI（终端界面）中运行 `/connect` 命令，选择 **OpenCode Zen** 并粘贴密钥。
3. 在 TUI 中运行 `/models` 查看推荐的模型列表。

系统将按请求收费，您可以随时为账户充值。

## 端点 (Endpoints)

您也可以通过以下 API 端点访问我们的模型。在 OpenCode 配置中，模型 ID 格式为 `opencode/<model-id>`。例如，使用 GPT 5.1 Codex 时，配置为 `opencode/gpt-5.1-codex`。

| 模型名称 | 模型 ID | 兼容 SDK 包 |
| --- | --- | --- |
| GPT 5.2 | gpt-5.2 | @ai-sdk/openai |
| GPT 5.1 Codex | gpt-5.1-codex | @ai-sdk/openai |
| Claude Sonnet 4.5 | claude-sonnet-4-5 | @ai-sdk/anthropic |
| Gemini 3 Pro | gemini-3-pro | @ai-sdk/google |
| Kimi K2 Thinking | kimi-k2-thinking | @ai-sdk/openai-compatible |
| Qwen3 Coder 480B | qwen3-coder | @ai-sdk/openai-compatible |

## 自动充值与限额 (Auto-reload & Monthly limits)

* **自动充值：** 如果余额低于 $5，Zen 将自动充值 $20（可自定义或禁用）。
* **月度限额：** 您可以为整个工作区或单个团队成员设置月度使用上限。

## 隐私 (Privacy)

所有模型均托管在美国。除限时免费模型（可能会收集数据用于模型改进）以及 OpenAI/Anthropic 的 30 天数据保留政策外，我们的供应商遵循**零保留政策**，不会将您的数据用于模型训练。

## 团队功能 (For Teams)

Zen 同样适用于团队协作：

* **角色管理：** 分为 管理员（Admin）和 成员（Member）。
* **模型访问控制：** 管理员可以为工作区启用或禁用特定模型。
* **自带密钥（BYOK）：** 您可以使用自己的 OpenAI 或 Anthropic API 密钥，此时 Token 费用由相应供应商直接结算，而非 Zen 扣费。

## 我们的目标 (Goals)

创建 OpenCode Zen 的初衷是：

1. 为编程代理找到最佳模型/服务商组合。
2. 确保最高质量，不为了降低成本而牺牲性能或路由到廉价服务商。
3. 透明计费：按成本价销售，仅加收处理费。
4. 无绑定：您可以将 Zen 与任何其他编程代理配合使用，也可以在 OpenCode 中使用任何其他服务商。
