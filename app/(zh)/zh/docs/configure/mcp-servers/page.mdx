# MCP 服务器 (MCP Servers)

模型上下文协议 (Model Context Protocol, MCP) 是一项开放标准，允许 OpenCode 连接到各种外部源和工具。

有关 MCP 的工作原理、架构和实际应用示例，请参阅我们的 **[MCP 综述指南](/zh/docs/mcp)**。

## 概览

OpenCode 允许你直接在 [opencode.jsonc](/zh/docs/config) 配置文件中配置 MCP 服务器。这些服务器可以提供额外的工具（Tools）、资源（Resources）和提示词（Prompts），供 AI 智能体在与你互动时使用。

## 配置

在你的 `opencode.jsonc` 文件的 `mcp` 部分添加服务器：

```json
{
  "mcp": {
    "server-name": {
      "type": "command",
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres", "postgresql://localhost/mydb"],
      "enabled": true,
      "env": {
        "MY_ENV_VAR": "value"
      }
    }
  }
}
```

### 属性

- `type`: 目前支持 `command`（本地脚本/命令）
- `command`: 要运行的命令（例如 `npx`, `python`, `node`, `docker`, `poetry`）
- `args`: 传递给命令的参数列表
- `enabled`: (可选) 布尔值，用于启用或禁用该服务器（默认为 `true`）
- `env`: (可选) 要传递给服务器进程的环境变量对象

## 修改配置

修改完成后，OpenCode 将会：
1. 检测到配置文件的更改。
2. 自动重启受影响的 MCP 服务器。
3. 更新该会话中可用的工具列表。

## 调试

你可以通过以下方式检查 MCP 服务器的状态：

1.  **TUI 界面**: 使用 `/mcp` 斜杠命令查看已连接的服务器及其工具。
2.  **日志**: OpenCode 会将 MCP 相关的错误记录到标准错误输出 (stderr)。

## [优先级顺序](/zh/docs/config#加载优先级顺序从先到后后者覆盖前者)

如果在多个配置文件中定义了同名的 MCP 服务器（例如在全局和项目配置中），则合并规则如下：

-   `enabled` 属性将被覆盖。
-   `env` 变量将被合并。
-   如果 `command` 不同，则更高优先级的配置将完全替换旧的配置。

## 故障排除

### 路径问题
在配置 `command` 时，尽量使用绝对路径或确保相关可执行文件（如 `node`, `python`）已在你的系统 `PATH` 中。

### 环境
如果服务器依赖特定的环境变量（如 API 密钥），请确保它们已在 `env` 键下正确定义。

### 权限
确保 OpenCode 具有运行指定命令及其关联脚本的必要权限。
