# 企业版 (Enterprise)

**在您的组织中安全地使用 OpenCode。**

OpenCode 企业版专为希望确保其代码和数据永远不离开其基础设施的组织而设计。它通过使用一个集成的中心化配置来实现这一点，该配置可与您的 SSO（单点登录）和内部 AI 网关结合。

> **注意：** OpenCode 不存储您的任何代码或上下文数据。

**开始使用 OpenCode 企业版：**

1. 在您的团队内部进行试用。
2. 联系我们讨论定价和实施方案。

## 试用 (Trial)

OpenCode 是开源的，且不存储您的任何代码或上下文数据，因此您的开发人员可以轻松地开始并进行内部试用。

## 数据处理 (Data handling)

OpenCode **不会**存储您的代码或上下文数据。所有处理均在本地进行，或通过直接 API 调用发送至您的 AI 服务商。

这意味着，只要您使用的是您信任的服务商或内部 AI 网关，您就可以安全地使用 OpenCode。

唯一的例外是可选的 `/share`（分享）功能。

### 分享对话 (Sharing conversations)

如果用户启用了 `/share` 功能，对话及其相关数据将被发送到我们在 `opencode.ai` 用于托管这些分享页面的服务中。

这些数据目前通过我们 CDN 的边缘网络提供服务，并在靠近用户的边缘节点进行缓存。

**我们建议在试用期间禁用此功能。**

**opencode.json**

```json
{
  "$schema": "https://opencode.ai/config.json",
  "share": "disabled"
}
```

[了解更多关于分享的信息。](/zh/docs/share)

## 代码所有权 (Code ownership)

您拥有 OpenCode 生成的所有代码。没有任何许可限制或所有权声明。

## 定价 (Pricing)

OpenCode 企业版采用**按席位（per-seat）计费**模式。如果您拥有自己的 LLM 网关，我们不会对使用的 Token 收费。有关定价和实施方案的更多详细信息，请联系我们。

## 部署 (Deployment)

当您完成试用并准备在组织中正式使用 OpenCode 时，可以联系我们讨论定价和实施细节。

### 中心化配置 (Central Config)

我们可以为您的整个组织设置 OpenCode 使用单一的中心化配置。该配置可以与您的 SSO 供应商集成，并确保所有用户仅访问您的内部 AI 网关。

### SSO 集成 (SSO integration)

通过中心化配置，OpenCode 可以与组织的 SSO 供应商集成进行身份验证。这允许 OpenCode 通过您现有的身份管理系统获取内部 AI 网关的凭据。

### 内部 AI 网关 (Internal AI gateway)

配合中心化配置，OpenCode 可以被配置为**仅使用**您的内部 AI 网关。您还可以禁用所有其他 AI 服务商，确保所有请求都通过组织批准的基础设施。

### 私有化部署 (Self-hosting)

虽然我们建议禁用分享页面以确保数据不外流，但我们也可以协助您在自己的基础设施上私有化部署这些页面。这目前在我们的路线图中，如果您感兴趣，请告知我们。

## 常见问题 (FAQ)

**什么是 OpenCode 企业版？**
OpenCode 企业版专为希望确保代码和数据留在内部基础设施的组织设计，通过中心化配置、SSO 集成和内部 AI 网关实现安全管控。

**如何开始使用 OpenCode 企业版？**
直接在团队中开始内部试用即可。默认情况下 OpenCode 不存储代码，非常容易起步。随后请联系我们讨论商务合作。

**企业版定价如何运作？**
按席位收费。如果您有自己的 LLM 网关，我们不收 Token 费。请联系我们获取定制报价。

**我的数据安全吗？**
是的。所有处理都在本地或通过直接 API 调用完成。配合中心化配置，数据将完全留在组织基础设施内。

**我们可以使用自己的私有 NPM 仓库吗？**
支持。OpenCode 通过 Bun 原生支持 `.npmrc` 文件。如果您使用 JFrog Artifactory、Nexus 等私有仓库，请确保开发人员在运行 OpenCode 前已完成身份验证。

**设置私有仓库身份验证：**

```bash
npm login --registry=https://your-company.jfrog.io/api/npm/npm-virtual/
```

这会生成包含验证信息的 `~/.npmrc`，OpenCode 会自动识别它。或者您可以手动配置 `.npmrc`。

> **警示：** 在运行 OpenCode 之前，必须先登录私有仓库。
