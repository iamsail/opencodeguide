# 模型提供商 (Providers)

OpenCode 通过 AI SDK 和 Models.dev 支持 75 种以上的 LLM 提供商，同时兼容本地模型运行。添加提供商需完成两步操作：一是使用 `/connect` 命令添加该提供商的 API 密钥；二是在 OpenCode 配置中完成提供商的相关配置。

## 凭证管理

使用 `/connect` 命令添加提供商的 API 密钥后，密钥会存储在 `~/.local/share/opencode/auth.json` 文件中。

## 配置说明

可通过 OpenCode 配置文件中的 `provider` 部分自定义提供商相关设置。

### 基础 URL 配置

如需使用代理服务或自定义端点，可通过 `baseURL` 选项自定义任意提供商的基础 URL。示例配置如下：

```json
{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "anthropic": {
      "options": {
        "baseURL": "https://api.anthropic.com/v1"
      }
    }
  }
}
```

## OpenCode Zen

OpenCode Zen 是由 OpenCode 团队提供的一系列模型集合，这些模型均经过测试验证，能与 OpenCode 良好兼容。

> [!TIP]
> 关于 OpenCode Zen 的更多高级配置和详细模型列表，请参阅 [OpenCode Zen 枢纽页](/zh/docs/zen)。

### 快速开始（推荐新手使用）

1. 在 TUI 中运行 `/connect` 命令，选择 opencode 选项，随后访问 opencode.ai/auth；
2. 登录账号，添加账单信息并复制 API 密钥；
3. 将 API 密钥粘贴到对应输入框；
4. 在 TUI 中运行 `/models` 命令，即可查看推荐的模型列表。

OpenCode Zen 的使用方式与其他提供商一致，且为可选功能。

## 各提供商详细配置

### 302.AI
1. 访问 302.AI 控制台，注册账号并生成 API 密钥；
2. 运行 `/connect` 命令，搜索并选择 302.AI；
3. 输入 302.AI 的 API 密钥；
4. 运行 `/models` 命令选择所需模型。

### Amazon Bedrock
#### 前置要求
需在 Amazon Bedrock 控制台的模型目录中，申请获取目标模型的访问权限。

#### 认证配置（任选一种方式）
1. **环境变量（快速开始）**
运行 OpenCode 时设置以下任一环境变量：
```bash
# 方式 1：使用 AWS 访问密钥
AWS_ACCESS_KEY_ID=XXX AWS_SECRET_ACCESS_KEY=YYY opencode

# 方式 2：使用命名 AWS 配置文件
AWS_PROFILE=my-profile opencode

# 方式 3：使用 Bedrock Bearer 令牌
AWS_BEARER_TOKEN_BEDROCK=XXX opencode
```
也可将环境变量添加到 bash 配置文件（~/.bash_profile）：
```bash
export AWS_PROFILE=my-dev-profile
export AWS_REGION=us-east-1
```

2. **配置文件（推荐）**
针对项目专属或持久化配置，可使用 opencode.json 文件：
```json
{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "amazon-bedrock": {
      "options": {
        "region": "us-east-1",
        "profile": "my-aws-profile"
      }
    }
  }
}
```

#### 可用配置选项
- `region`：AWS 区域（例如 us-east-1、eu-west-1）；
- `profile`：来自 ~/.aws/credentials 的 AWS 命名配置文件；
- `endpoint`：VPC 端点的自定义端点 URL（等同于通用的 baseURL 选项）。

> [!NOTE]
> 配置文件中的选项优先级高于环境变量。

#### 高级配置：VPC 端点
若使用 VPC 端点访问 Bedrock，配置示例如下：
```json
{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "amazon-bedrock": {
      "options": {
        "region": "us-east-1",
        "profile": "production",
        "endpoint": "https://bedrock-runtime.us-east-1.vpce-xxxxx.amazonaws.com"
      }
    }
  }
}
```

#### 认证方式
- `AWS_ACCESS_KEY_ID`/`AWS_SECRET_ACCESS_KEY`：在 AWS 控制台创建 IAM 用户并生成访问密钥；
- `AWS_PROFILE`：使用 ~/.aws/credentials 中的命名配置文件，需先通过 `aws configure --profile my-profile` 或 `aws sso login` 完成配置；
- `AWS_BEARER_TOKEN_BEDROCK`：从 Amazon Bedrock 控制台生成长期 API 密钥；
- `AWS_WEB_IDENTITY_TOKEN_FILE`/`AWS_ROLE_ARN`：适用于 EKS IRSA（服务账号的 IAM 角色）或其他支持 OIDC 联合身份验证的 Kubernetes 环境，相关环境变量会由 Kubernetes 在使用服务账号注解时自动注入。

#### 认证优先级
1. Bearer 令牌（通过 `/connect` 命令或 `AWS_BEARER_TOKEN_BEDROCK` 环境变量设置）；
2. AWS 凭证链（配置文件、访问密钥、共享凭证、IAM 角色、Web 身份令牌（EKS IRSA）、实例元数据）。

> [!NOTE]
> 当设置了 Bearer 令牌时，其优先级高于所有 AWS 凭证方式（包括已配置的配置文件）。

#### 最终步骤
运行 `/models` 命令选择所需模型。

### Anthropic
1. 推荐注册 Claude Pro 或 Max 账号；
2. 注册完成后，运行 `/connect` 命令并选择 Anthropic；
3. 选择认证方式：
    - **Claude Pro/Max**：选择该选项后，浏览器会自动打开并要求完成身份验证；
    - **Create an API Key**：若无 Pro/Max 订阅，可选择该选项，浏览器打开后登录 Anthropic 并获取验证码，将验证码粘贴到终端；
    - **Manually enter API Key**：若已有 API 密钥，直接粘贴到终端；
4. 运行 `/models` 命令，即可使用所有 Anthropic 模型。

### Azure OpenAI
#### 注意事项
若遇到“I’m sorry, but I cannot assist with that request”错误，可尝试在 Azure 资源中将内容筛选器从 DefaultV2 改为 Default。

#### 配置步骤
1. 访问 Azure 门户，创建 Azure OpenAI 资源，需获取：
    - 资源名称：将成为 API 端点的一部分（格式：https://RESOURCE_NAME.openai.azure.com/）；
    - API 密钥：资源中的 KEY 1 或 KEY 2；
2. 前往 Azure AI Foundry 部署模型（部署名称必须与模型名称一致，否则 OpenCode 无法正常工作）；
3. 运行 `/connect` 命令，搜索并选择 Azure；
4. 输入 API 密钥；
5. 设置资源名称为环境变量：
```bash
AZURE_RESOURCE_NAME=XXX opencode
```
或添加到 bash 配置文件（~/.bash_profile）：
```bash
export AZURE_RESOURCE_NAME=XXX
```
6. 运行 `/models` 命令选择已部署的模型。

### Cloudflare AI Gateway
Cloudflare AI Gateway 支持通过统一端点访问 OpenAI、Anthropic、Workers AI 等多个提供商的模型，且通过统一计费无需为每个提供商单独配置 API 密钥。

#### 配置步骤
1. 访问 Cloudflare 控制台，导航至 AI > AI Gateway，创建新网关；
2. 设置环境变量（添加到 ~/.bash_profile）：
```bash
export CLOUDFLARE_ACCOUNT_ID=你的 32 位账号 ID
export CLOUDFLARE_GATEWAY_ID=你的网关 ID
```
3. 运行 `/connect` 命令，搜索并选择 Cloudflare AI Gateway；
4. 输入 Cloudflare API 令牌（也可通过环境变量设置：`export CLOUDFLARE_API_TOKEN=你的 API 令牌`）；
5. 运行 `/models` 命令选择所需模型。

#### 可选配置：通过配置文件添加模型
```json
{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "cloudflare-ai-gateway": {
      "models": {
        "openai/gpt-4o": {},
        "anthropic/claude-sonnet-4": {}
      }
    }
  }
}
```

### GitLab Duo
GitLab Duo 通过 GitLab 的 Anthropic 代理，提供支持原生工具调用功能的 AI 驱动智能聊天服务。

#### 配置步骤
1. 运行 `/connect` 命令并选择 GitLab；
2. 选择认证方式：
    - **OAuth（推荐）**：选择后浏览器会自动打开以完成授权；
    - **Personal Access Token**：
      1. 前往 GitLab 用户设置 > Access Tokens；
      2. 点击“Add new token”，名称设为 OpenCode，权限范围勾选 api；
      3. 复制生成的令牌（以 glpat- 开头）并粘贴到终端；
3. 运行 `/models` 命令查看可用模型。

#### 可用模型（基于 Claude）
- `duo-chat-haiku-4-5`（默认）：响应快速，适用于简单任务；
- `duo-chat-sonnet-4-5`：性能均衡，适用于大多数工作流；
- `duo-chat-opus-4-5`：功能最强，适用于复杂分析场景。

#### 自托管 GitLab 配置
1. 直接运行命令：
```bash
GITLAB_INSTANCE_URL=https://gitlab.company.com GITLAB_TOKEN=glpat-xxxxxxxxxxxxxxxxxxxx opencode
```
2. 或添加到 bash 配置文件（~/.bash_profile）：
```bash
export GITLAB_INSTANCE_URL=https://gitlab.company.com
export GITLAB_TOKEN=glpat-xxxxxxxxxxxxxxxxxxxx
```

#### 高级配置
通过 opencode.json 自定义配置：
```json
{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "gitlab": {
      "options": {
        "instanceUrl": "https://gitlab.com",
        "featureFlags": {
          "duo_agent_platform_agentic_chat": true,
          "duo_agent_platform": true
        }
      }
    }
  }
}
```

#### 可选：GitLab API 工具
如需访问 GitLab 工具（合并请求、问题、流水线、CI/CD 等），需添加插件配置：
```json
{
  "$schema": "https://opencode.ai/config.json",
  "plugin": ["@gitlab/opencode-gitlab-plugin"]
}
```
该插件提供全面的 GitLab 仓库管理功能，包括合并请求审核、问题跟踪、流水线监控等。

### Google Vertex AI
#### 前置要求
- 拥有已启用 Vertex AI API 的 Google Cloud 项目；
- 在 Google Cloud 控制台的 Model Garden 中，确认目标模型在所在区域可用。

#### 配置步骤
1. 设置必要的环境变量：
    - `GOOGLE_CLOUD_PROJECT`：Google Cloud 项目 ID；
    - `VERTEX_LOCATION`（可选）：Vertex AI 区域（默认值为 global）；
2. 选择认证方式：
    - 设置 `GOOGLE_APPLICATION_CREDENTIALS`：指向服务账号 JSON 密钥文件的路径；
    - 通过 gcloud CLI 认证：运行 `gcloud auth application-default login`；
3. 运行 OpenCode 时加载环境变量：
```bash
GOOGLE_APPLICATION_CREDENTIALS=/path/to/service-account.json GOOGLE_CLOUD_PROJECT=your-project-id opencode
```
或添加到 bash 配置文件（~/.bash_profile）：
```bash
export GOOGLE_APPLICATION_CREDENTIALS=/path/to/service-account.json
export GOOGLE_CLOUD_PROJECT=your-project-id
export VERTEX_LOCATION=global
```

> [!TIP]
> global 区域可提高可用性并减少错误，且无需额外费用；若有数据驻留要求，可使用区域端点（例如 us-central1）。

4. 运行 `/models` 命令选择所需模型。

### llama.cpp（本地模型）
通过 llama.cpp 的 llama-server 工具，可配置 OpenCode 使用本地模型：
```json
{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "llama.cpp": {
      "npm": "@ai-sdk/openai-compatible",
      "name": "llama-server (local)",
      "options": {
        "baseURL": "http://127.0.0.1:8080/v1"
      },
      "models": {
        "qwen3-coder:a3b": {
          "name": "Qwen3-Coder: a3b-30b (local)",
          "limit": {
            "context": 128000,
            "output": 65536
          }
        }
      }
    }
  }
}
```

#### 配置说明
- `llama.cpp`：自定义提供商 ID（可任意命名）；
- `npm`：指定该提供商使用的包（OpenAI 兼容 API 均使用 @ai-sdk/openai-compatible）；
- `name`：UI 中显示的提供商名称；
- `options.baseURL`：本地服务器的端点；
- `models`：模型 ID 与配置的映射，模型名称会显示在模型选择列表中。

#### 配置步骤

export AICORE_SERVICE_KEY='{"clientid":"...","clientsecret":"...","url":"...","serviceurls":{"AI_API_URL":"..."}}'

AICORE_DEPLOYMENT_ID=your-deployment-id AICORE_RESOURCE_GROUP=your-resource-group opencode

## 自定义提供商（OpenAI 兼容）

对于未在 `/connect` 命令列表中列出的 OpenAI 兼容提供商，可按以下步骤添加：

### 配置步骤

1. 运行 `/connect` 命令，滚动至底部选择“Other”；
2. 输入自定义提供商的唯一 ID（需牢记，将用于配置文件）；
3. 输入该提供商的 API 密钥（仅存储凭证，需后续在配置文件中完成配置）；
4. 在项目目录中创建或更新 opencode.json 文件：

```json
{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "myprovider": { // 与步骤 2 中输入的提供商 ID 一致
      "npm": "@ai-sdk/openai-compatible",
      "name": "My AI Provider Display Name", // UI 中显示的名称
      "options": {
        "baseURL": "https://api.myprovider.com/v1", // 提供商 API 端点
        "apiKey": "{env:ANTHROPIC_API_KEY}", // 可选，若不使用 auth 存储
        "headers": { // 可选，自定义请求头
          "Authorization": "Bearer custom-token"
        }
      },
      "models": {
        "my-model-name": {
          "name": "My Model Display Name", // 模型显示名称
          "limit": { // 可选，模型令牌限制
            "context": 200000, // 模型支持的最大输入令牌数
            "output": 65536 // 模型支持的最大输出令牌数
          }
        }
      }
    }
  }
}
```

5. 运行 `/models` 命令，自定义提供商及模型会出现在选择列表中。

## 故障排除

1. **检查认证配置**：运行 `opencode auth list` 查看提供商凭证是否已添加（不适用于依赖环境变量认证的提供商，如 Amazon Bedrock）；
2. **自定义提供商配置检查**：
    - 确保 `/connect` 命令中使用的提供商 ID 与配置文件中的 ID 一致；
    - 为提供商选择正确的 npm 包（例如 Cerebras 使用 @ai-sdk/cerebras，其他 OpenAI 兼容提供商使用 @ai-sdk/openai-compatible）；
    - 确认 `options.baseURL` 字段中配置的 API 端点正确。
