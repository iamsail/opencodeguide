# OpenCode CLI 文档

OpenCode CLI 提供了丰富的命令行选项和子命令，用于与 OpenCode 进行程序化交互。  
**不带任何参数**直接运行时，默认会启动 **TUI（终端用户界面）**：

```bash
opencode
```

快速示例（非交互模式直接提问）：

```bash
opencode run "解释一下 JavaScript 中的闭包是如何工作的"
```

## tui - 启动终端用户界面

```bash
opencode [project]
```

- `[project]`：可选，指定项目路径

常用选项（Flags）：

- `--continue` / `-c`：继续上一次会话
- `--session` / `-s`：指定会话 ID 继续
- `--prompt`：指定使用的提示词（prompt）
- `--model` / `-m`：指定模型，格式 `provider/model`
- `--agent`：指定使用的 Agent（代理）
- `--port`：监听端口
- `--hostname`：监听主机名

## 主要命令一览

### agent - 管理 Agents（代理）

```bash
opencode agent [子命令]
```

- `opencode agent create`  
  交互式创建自定义 Agent（设置 system prompt 和工具）

- `opencode agent list`  
  列出所有可用 Agent

### auth - 管理模型提供商认证

```bash
opencode auth [子命令]
```

凭证存储在 `~/.local/share/opencode/auth.json`，也支持读取环境变量和项目 `.env` 文件。

- `opencode auth list` / `ls`  
  列出已认证的提供商

- `opencode auth logout`  
  注销/清除某个提供商的凭证

### github - 管理 GitHub 自动化代理

- `opencode github install`  
  在当前仓库安装 GitHub Agent（会设置 GitHub Actions 工作流）

- `opencode github run`  
  运行 GitHub Agent（常用于 Actions 环境）

  选项：
  - `--event`：模拟的 GitHub 事件
  - `--token`：GitHub Personal Access Token

### mcp - 管理 Model Context Protocol 服务器

- `opencode mcp add` → 交互式添加本地/远程 MCP 服务器
- `opencode mcp list` / `ls` → 列出已配置的 MCP 服务器及连接状态
- `opencode mcp auth [name]` → 对支持 OAuth 的 MCP 服务器进行认证
- `opencode mcp auth list/ls` → 列出支持 OAuth 的服务器及认证状态
- `opencode mcp logout [name]` → 移除 OAuth 凭证
- `opencode mcp debug <name>` → 调试 OAuth 连接问题

### models - 列出可用模型

```bash
opencode models [provider]
```

示例：
```bash
opencode models anthropic
opencode models --refresh          # 从 models.dev 刷新缓存
opencode models --verbose          # 显示更多信息（含价格等元数据）
```

### run - 非交互模式快速运行

```bash
opencode run [消息内容...]
```

示例：
```bash
opencode run "Go 语言中的 context 具体有什么用？"
```

支持附加到已运行的服务器（避免冷启动）：
```bash
opencode run --attach http://localhost:4096 "解释 JavaScript async/await"
```

常用选项：

- `--command`：把消息当作命令参数执行
- `--continue` / `-c`：继续上一次会话
- `--session` / `-s`：指定会话 ID
- `--share`：分享本次会话
- `--model` / `-m`：指定模型
- `--agent`：指定 Agent
- `--file` / `-f`：附加文件
- `--format`：输出格式（default / json）
- `--title`：会话标题
- `--attach`：附加到已有服务器 URL
- `--port`：本地服务器端口

### serve - 启动无头 HTTP API 服务器

```bash
opencode serve
```

选项：
- `--port` / `--hostname` / `--mdns` / `--cors`

可设置环境变量 `OPENCODE_SERVER_PASSWORD` 启用基本认证（默认用户名为 `opencode`）。

### web - 启动带网页界面的服务器

```bash
opencode web
```

自动打开浏览器，选项同 `serve`。

### session - 管理会话

- `opencode session list`  
  列出所有会话

  选项：
  - `--max-count` / `-n`：显示最近 N 条
  - `--format`：table 或 json

### stats - 显示使用统计

```bash
opencode stats
```

选项：
- `--days`：最近 N 天
- `--tools` / `--models`：显示工具/模型明细
- `--project`：按项目过滤

### export & import - 会话导出/导入

```bash
opencode export [sessionID]          # 导出为 JSON
opencode import session.json         # 从文件导入
opencode import https://opncd.ai/s/xxx  # 从分享链接导入
```

### acp - 启动 Agent Client Protocol 服务器

```bash
opencode acp
```

通过 stdin/stdout 使用 ndjson 通信。

选项：`--cwd` / `--port` / `--hostname`

### uninstall & upgrade

- `opencode uninstall`  
  卸载（可加 `--keep-config` `--keep-data` `--dry-run` `--force`）

- `opencode upgrade [版本号]`  
  升级（不带参数 → 最新版；可指定版本如 v0.1.48）

  选项：`--method`（curl/npm/pnpm/bun/brew）

## 全局选项（任何命令都可用）

- `--help` / `-h`
- `--version` / `-v`
- `--print-logs`
- `--log-level` （DEBUG / INFO / WARN / ERROR）
