# 迁移指南 (Migrating to 1.0)

## 这是什么？
迁移指南专为从 OpenCode 旧版本（0.x 或 Beta 版）升级到 1.0 正式版的用户准备。它详细记录了版本之间的重大变更（Breaking Changes）、配置文件的格式更新以及新老功能的映射关系。

## 什么时候你会用到它？
- 当你决定升级到 OpenCode 1.0 版本，且之前的项目中已有旧版配置文件时。
- 当你发现升级后，某些自定义脚本或快捷键失效时。
- 当你需要了解 1.0 版本带来了哪些核心架构的改进和新特性时。

## 核心概念
- **破坏性变更 (Breaking Changes)**：导致旧有配置或使用方式无法继续工作的修改，必须手动调整。
- **配置迁移工具**：可能提供的自动化脚本，用于将旧版配置转换为新版格式。
- **完全兼容性**：1.0 版本承诺的稳定 API 列表，未来版本将保持向后兼容。
