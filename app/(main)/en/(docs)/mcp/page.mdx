# Model Context Protocol (MCP) in OpenCode — Complete Guide

Model Context Protocol (MCP) is the open standard that enables OpenCode to connect directly with your **tools, data, and context**. Unlike closed plugin systems, MCP creates a universal interface for AI to query your code, databases, and local environment securely.

If you are looking to **automate browser testing**, **manage GitHub issues**, or **connect internal APIs** to OpenCode, MCP is the answer.

---

## What is MCP (Model Context Protocol)?

MCP is a standardized protocol that connects AI models to external tools and context.

Think of it as **USB-C for AI applications**. Instead of every AI editor building its own custom integrations for GitHub, PostgreSQL, or Slack, MCP provides a single standard. Any tool that speaks MCP can immediately work with OpenCode.

> **Need deep technical details?**  
> For the JSON-RPC spec, schema details, and protocol internals, see our [**MCP Technical Deep Dive**](/en/mcp-explained).

---

## Why OpenCode Uses MCP

Traditional editors use isolated plugins or scripts. OpenCode adopts MCP because it offers:

1.  **Universality**: The same MCP server you write for OpenCode works with other MCP-compliant clients (like Claude Desktop).
2.  **Context-Awareness**: Tools aren't just "commands"; they provide context that the AI can read before acting.
3.  **Security**: You control exactly what the AI can see and do via your local MCP servers, keeping data private.

---

## How MCP Works in OpenCode

Architecture:

```text
User Request
     |
     v
OpenCode (Client)
     |
     +--- connects via stdio/http ---+
     |                               |
MCP Server A                     MCP Server B
(e.g., Filesystem)               (e.g., GitHub)
     |                               |
     v                               v
Local Files                      Issues/PRs
```

When you ask OpenCode to "Check the latest PR", it doesn't hallucinate. It asks the **GitHub MCP Server** to fetch the real data, then uses that grounded context to answer you.

## How to Set Up MCP in OpenCode

Enabling MCP is a 3-step process.

### 1. Install OpenCode CLI (Optional)
Ensure you are running the latest version of OpenCode.

### 2. Configure Your Servers
Edit your `config.json` to tell OpenCode where your MCP servers connect.

### 3. Verify Connection
Restart OpenCode. You should see "MCP Servers Connected" in the status bar or logs.

> **Full Setup Guide**  
> For detailed installation steps and troubleshooting, see [**MCP Configuration Guide**](/en/mcp-configuration).

## MCP Configuration Examples

Here are quick copy-paste snippets for common setups.

### Playwright (Browser Automation)

```json
{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-playwright"]
    }
  }
}
```

### Generic Local Server (Python)

```json
{
  "mcpServers": {
    "my-tool": {
      "command": "python",
      "args": ["/path/to/server.py"]
    }
  }
}
```

## Common MCP Use Cases

**1. "Fix the test failure in the browser"**  
Using **Playwright MCP**, OpenCode can launch a browser, reproduce the error, capture the console log, and suggest a fix.

**2. "Summarize the last 5 user issues related to login"**  
Using **GitHub MCP**, OpenCode fetches real issue descriptions and synthesizes a report.

**3. "Validate this SQL query against the schema"**  
Using **Postgres MCP**, OpenCode checks your query columns against the live database schema.
