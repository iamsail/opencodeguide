# Oh My OpenCode: The Definitive Guide to Multi-Agent Workflows

**Oh My OpenCode** is an opinionated setup for OpenCode that helps developers run **multi-agent coding workflows** using Claude and MCP tools — without reinventing everything from scratch.

It’s not a separate product.  
It’s a **battle-tested preset** combining agents, rules, and MCP integrations for real-world development tasks.

---

## What Is Oh My OpenCode?

Oh My OpenCode is a **curated configuration** for OpenCode that focuses on:

- Multi-agent orchestration
- Long-running coding tasks
- MCP-first tool integration
- Practical, production-oriented workflows

Think of it as the difference between:
- *“I can use OpenCode”*  
- and *“I can ship with OpenCode efficiently.”*

## Why Developers Use Oh My OpenCode

Developers adopt Oh My OpenCode when:

- A single assistant is not enough
- Tasks require planning → coding → reviewing → tooling
- Browser automation or repo access is involved
- Context management becomes a bottleneck

### Key advantages

- **Multi-agent by default**  
  Planner, Coder, Reviewer, Tool agents work together.

- **Opinionated, not bloated**  
  Sensible defaults that reduce setup friction.

- **MCP-first architecture**  
  Tools like Playwright, GitHub, filesystem are first-class citizens.

- **Designed for real workflows**  
  Feature development, refactors, debugging, code reviews.

## When Should You Use Oh My OpenCode?

Use it when:

- Tasks are complex or long-running
- Multiple perspectives improve output quality
- Tool access is required
- You want consistency across runs

Avoid it when:
- Tasks are trivial
- Latency matters more than quality
- Context size is extremely constrained
