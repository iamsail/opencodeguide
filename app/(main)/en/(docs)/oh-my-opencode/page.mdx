# Oh My OpenCode

**Oh My OpenCode** is an AI-first coding workflow designed for developers who want to build faster, think clearer, and reduce context switching when working with modern AI tools.

Instead of chatting with an AI in isolation, OpenCode helps you **structure coding tasks as workflows**, combining prompts, agents, and tools into a repeatable system.

---

## What Is Oh My OpenCode?

Oh My OpenCode is a **workflow-oriented AI coding system**.

It focuses on:

- Turning prompts into reusable workflows  
- Coordinating multiple AI agents for complex tasks  
- Integrating tools (like testing, browsers, or MCPs) into the coding loop  

Rather than replacing your editor, OpenCode **augments how you think and work as a developer**.

---

## Who Is Oh My OpenCode For?

Oh My OpenCode is built for developers who:

- Build frequently with AI-assisted coding tools  
- Use VS Code, Cursor, or similar AI-enhanced editors  
- Want more structure than “chat-based prompting”  
- Care about repeatability, not one-off answers  

Typical users include:

- Indie developers  
- AI-first engineers  
- Workflow-driven builders  
- Developers comparing OpenCode vs Claude Code  

---

## What Problems Does It Solve?

### 1. Context Switching Fatigue
Jumping between chat windows, editors, docs, and tools breaks flow.  
OpenCode centralizes reasoning, execution, and iteration.

### 2. One-Off Prompting
Chat prompts disappear.  
Workflows **persist, evolve, and improve over time**.

### 3. Unstructured AI Usage
Instead of asking “what should I ask next?”, workflows encode **what happens next by design**.

---

## How Oh My OpenCode Works (High-Level)

At a high level, OpenCode is built around three core ideas:

### Workflows
A workflow defines **what steps happen, in what order, and why**.

### Agents
Agents handle specific responsibilities such as:
- Reasoning
- Refactoring
- Testing
- Reviewing output

### Tool Integration
OpenCode connects AI reasoning with real actions:
- File system
- Test runners
- Browsers
- MCP-based tools (e.g. Playwright)

This creates a closed loop between **thinking → doing → verifying**.
